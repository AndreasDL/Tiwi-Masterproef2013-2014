Automatische testbed monitoring voor jFed - Fed4Fire

Een masterproef door Andreas De Lille

Doelstellingen

Deze masterproef zal in de context van het Europese onderzoeksproject Fed4FIRE de tool jFed verder uitwerken.Het voornaamste doel is dat de onderzoeker via zijn primaire gebruikersinterface op de hoogte gebracht wordt van eventuele storingen. Deze storingen kan men opdelen in 2 categorieën, enerzijds op het niveau van testbeds en anderzijds op het niveau van componenten eigen aan een experiment.

In een eerste deel zal een api uitgewerkt worden. Deze zal via een webservice toegang geven tot de databank waar de monitoring informatie opgeslagen is. Deze api moet de huidige ad-hoc oplossing efficiënt vervangen. De api zal waarschijnlijk uitgeschreven worden in php en aangeboden worden met een restful service over http. Ook moet deze informatie actueel gehouden worden. Momenteel gebeurt dit met shell scripts. Dat kan uiteraard efficiënter.

Na het uitwerken van deze api zal er een webinterface uitgewerkt worden. Deze website zal de huidige sites vervangen. Er zal een algemeen overzicht gegeven worden. Ook gedetailleerde weergeven van een test of een testbed moet mogelijk zijn. Daarnaast zal er ook een gui component voor jFed gemaakt worden om deze informatie weer te geven.

Een eventuele uitbereiding is verder gaan dan monitoring op testbed niveau. Men kan ook nodes in een test monitoren.
