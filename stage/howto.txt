Wat:
layout => intelliJ idea project
De gui in een standalone project.

jFed Andreas => intelliJ idea project
mijn deel gui in de originel jFed code verwerkt.

Waar (in jFed app):
start jfed-advancedgui jar (in target map van advanced gui package)
login
create slice
4e paneel met 'RightClick Pane' op openen


Acties:

dubbel click => nieuwe node toevoegen
rechter drag => paneel verschuiven
scroll => zoom
linker drag van ene naar andere node => nieuwe verbinding toevoegen (zichtbaar na plaatsing).
rechter klik => rechterklik menu => delete & properties (nieuw venster).
linkerdrag op wit linker bovenhoekje van node => node verplaatsen.


Know issues:

Het toevoegen van een verbinding kan soms fouten geven.
Het mouseup event wordt altijd naar de node gestuurd waar de mousedown gebeurde.
Ik heb bijgevolg het paneel laten 'zoeken' naar de node op basis van coordinaten, 
maar het NodePane neemt de volledig grootte in van de stage. Dit is gecorrigeerd met constanten en werkte op  mijn laptop wel.

css file van NodePane wordt niet ingeladen. Ik heb dit op verschillende manieren geprobeerd, maar het is niet gelukt.
Ik heb de code doorgestuurd naar Cedric en hij heeft een aantal aanpassingen gedaan, waardoor het bij hem wel zichtbaar was.
(de achtergrond kleur van het NodePane is normaal niet wit, maar heeft een grijs-zwarte linear-gradient).
Dezelfde css laadt wel in mijn standalone project.
